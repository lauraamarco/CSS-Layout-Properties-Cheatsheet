Display

The Display element specifies whether an HTML element is treated as a block or an inline element. It determines the layout behavior of an element and how it interacts with other elements around it.
Here are some common values for the display property:
Value Description Added to the element or it's container?
block They create a line break after the element and occupy the full width of their parent container. Element
inline They don’t create line breaks and occupy only the necessary width and height based on their content. Element
inline-block They behave like inline elements but allow you to set width, height, padding, and margins. Element
none These elements won't be rendered and are completely hidden from the page. They don’t take up any space in the document flow. It is commonly used with JavaScript to hide or show elements without deleting and recreating them. Element
flex They create a flex container, allowing you to build flexible and responsive layouts. Flex items inside the container can be positioned and resized using flexbox properties. More about CSS flexbox properties here (Codecademy) and here (MDN Docs). Container
grid They create a grid container that enables you to define rows and columns, creating a two-dimensional grid system. Child elements inside the container can be placed in specific cells and adjusted for size and alignment. More about CSS grid properties here (Codecademy) and here (MDN Docs). Container
HTML element has a default display value, depending on what type of element it is. The default display value for most elements is block or inline . The display property is used to change the default display behavior of HTML elements.
  • A block-level element always starts on a new line and takes up the full width available.
  • Some examples for elements that have a block display by default are <div>, <h1> - <h6>, <p>, <form>, <header>, <footer>, <section>.
  • An inline element does not start on a new line and only takes up as much width as necessary.
  • These elements will show with inline display by default: <span>, <a>, <strong>, <em>, <img>.
For more details on the Display property see here: Codecademy Docs - Display.

Position

The position property defines how elements are placed within the document.
Here are some common values for the position property:
Value Description Remains in the normal document flow? Needs offset properties (top, right, bottom, left)?
static Default value. The element appears in the actual location where it’s located in the markup.The offset properties or the z-index have no effect. Yes No
relative The element does not change positions unless you add position properties (top, right...). It also creates a reference position for absolute positioned child elements. Yes Yes
absolute The element is partially removed from the document flow and it’s positioned in relation to its closest relative positioned ancestor element. No space is created for the element in the page layout. The margins of absolutely positioned boxes do not collapse with other margins. No Yes
fixed It is removed from the document flow just like the above. However, unlike the absolute positioned elements, which are relative to an ancestor element, the fixed elements are always relative to the document (which is the viewport in the case of visual media). No Yes
sticky This value is a mix between the relative and fixed positions. The element is treated as relative until a specific point or threshold is met, at which point the element is then treated as fixed. Yes/No Yes
For more details on the Position property see here: Codecademy Docs - Position and MDN Docs - Position.

Floats

The float property moves an element to the left or right within its containing element, outside of the default positioning (determined by Position properties).
Here are some common values for the float property:
Value Description
left Makes the element float on the left of its container.
right Makes the element float on the right of its container.
none The element does not float and remains in the normal document flow.
inline-start The element must float on the start side of its containing block. That is the left side with ltr scripts, and the right side with rtl scripts.
inline-end The element must float on the end side of its containing block. That is the right side with ltr scripts, and the left side with rtl scripts.
This property is commonly used for wrapping text around an image; however, for layout tasks it's often better to use CSS Grid or Flexbox.
  • Float can only be applied to elements whose position is static or relative.
  • Floated elements are removed from the normal document flow, so other elements behave as if the floated element isn't there. Floated items usually need an explicit width; otherwise they can expand to the full width of their container.
  • When an element is floated, surrounding content (text and inline elements) will wrap around it. To prevent wrapping you can use the clear property on following elements.
For more details on the Position property see here: Codecademy Docs - Floats and MDN Docs - Position.

Box model

The CSS box model describes how every element is represented as a rectangular box: content, padding, border and margin. Understanding the box model is key to controlling spacing and sizing.
Common parts of the box model:
Part Description
content The area where text and images appear. Its size is controlled by the element's width and height (unless box-sizing applies).
padding Space between content and border. Padding increases the element's inner spacing and (depending on box-sizing) can affect total size. Use this to create space inside an element.
border The line surrounding padding and content. Border width and style contribute to the element's visual size unless using box-sizing: border-box.
margin The outermost space between the element and its neighbors. Margins are transparent and do not have a background color. Use this to create space outside an element.
Note: using box-sizing: border-box makes the width and height include padding and border, which often simplifies layout calculations.
For more on the box model see the MDN Box Model or Codecademy Box Model.
